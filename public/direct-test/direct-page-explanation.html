Note: Do not use relative links (like ./test-page.html) when linking between test pages. Always use absolute paths starting with /direct-test/.

Why This Is Needed

The standard React application uses client-side routing which intercepts URL changes. This means that when you try to navigate to a URL like /test/page.html, the React router catches it and tries to find a matching route in the React application.

The /direct-test/ URLs bypass this behavior by being handled directly by the Express server before React's router can intercept them.
